About

I'm a software engineer based in Brooklyn. I studied Computer Science at Skidmore College, with a minor in Management and Business.

My work is mostly backend-focused: APIs, infrastructure, deployment, GitHub workflows, and the systems around how code gets reviewed, shipped, and maintained. The languages I reach for most are C, Go, and TypeScript. I like typed languages for their compile-time checks, clearer interfaces, and the readability they bring to larger codebases.

Right now, I'm doing backend contract work for an early-stage startup, helping with API design, deployment reliability, CI/CD guardrails, and code review workflow.

I use AI tools in my daily workflow, but I don't see them as a replacement for engineering judgment. They make it easier to produce code quickly, which makes the surrounding standards more important: tests, review habits, project structure, consistency, and maintainability.

Outside of software, I enjoy reading, cooking, and game design.
